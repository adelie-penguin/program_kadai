# 並列処理のテストプログラム

コンパイル・実行コマンドは，各ファイルの先頭部分に記述されています．

| ファイル名 |  |
----|---- 
| cuda.cu | CUDAで並列処理(CPU-GPU間のデータ転送あり) |
| cuda2.cu | CUDAで並列処理(GPU側から表示) |
| pthread.c | pthreadで並列処理 |
| mpi.c | MPIで並列処理 |
| openmp.c |  |
| makefile | cuda2.cuのみをコンパイル可能 |


## 利用上の注意

- makefileは研究室のwss11～wss15環境以外だと正しく動作しませんので注意してください．
- mpi.c の実行コマンドは実行環境に合わせてコマンド引数を編集してください．
- 配布プログラムは実行結果が一意に定まりません．何度か実行して確認しておきましょう．
- 再配布は構いませんが，実行結果には責任を持ちませんのであしからず．

## makefileの補足説明

研究室環境でcuda2.cuをコンパイルするために下記のコマンドを実行します．
```
cuda2 : $(SRC).cu
	nvcc -o $@ $< $(CFLAG) $(GFLAG) 
```
`$@`や`$<`はmakefile特有の構文です．上の行にある`:`前後の内容で置き換えられます．  
例えばmakefileをwss15で実行すると，コンパイルコマンドは下記になります．
```
  nvcc -o cuda2 cuda2.cu -lm -O3 -arch=sm_70 --ptxas-options=-v
```

ファイル内の分岐処理は，コンパイル時にロングインしているコンピュータで利用可能なGPUのバージョンを`GFLAG`に設定するためのものです．  
分岐処理でwss13が特別扱いなのは，Jetsonでも同じmakefileを使いたかったためです．
